Задача 

разработать пакет для Flutter под названием typed_prefs, который предоставляет типизированный интерфейс для работы с SharedPreferences и flutter_secure_storage.

Принцип работы пользователь описывает все нужные ему ключи в классе пример:

enum PreferenceStorage {
  shared,
  secure,
}

class PreferenceKey<T> {
  final String key;
  final PreferenceStorage storage;
  final T? defaultValue;
  final String description;

  const PreferenceKey({
    required this.key,
    required this.storage,
    this.defaultValue,
    required this.description,
  });
}

class AppPreferencesKeys {
  static const themeMode = PreferenceKey<String>(
    key: 'theme_mode',
    storage: PreferenceStorage.shared,
    defaultValue: 'system',
    description: 'Theme mode of the application',
  );

  static const biometricsEnabled = PreferenceKey<bool>(
    key: 'biometrics_enabled',
    storage: PreferenceStorage.secure,
    defaultValue: false,
    description: 'Whether biometric auth is enabled',
  );

  static const vaultKey = PreferenceKey<String>(
    key: 'vault_key',
    storage: PreferenceStorage.secure,
    description: 'Encryption key for vault',
  );
}

что помечено PreferenceStorage.secure, то данные сохраняются в flutter_secure_storage, а что помечено PreferenceStorage.shared, то данные сохраняются в SharedPreferences.

пример реализации:

class PreferencesStorageRouter {
  final SharedPreferences shared;
  final FlutterSecureStorage secure;

  PreferencesStorageRouter({
    required this.shared,
    required this.secure,
  });

  Future<void> write<T>(PreferenceKey<T> key, T value) async {
    if (key.storage == PreferenceStorage.secure) {
      await secure.write(key: key.key, value: value.toString());
    } else {
      if (value is bool) {
        await shared.setBool(key.key, value);
      } else if (value is int) {
        await shared.setInt(key.key, value);
      } else if (value is double) {
        await shared.setDouble(key.key, value);
      } else if (value is String) {
        await shared.setString(key.key, value);
      } else {
        throw UnsupportedError('Type not supported');
      }
    }
  }

  Future<T?> read<T>(PreferenceKey<T> key) async {
    if (key.storage == PreferenceStorage.secure) {
      final value = await secure.read(key: key.key);
      return value as T?;
    }

    final value = shared.get(key.key);
    return value as T?;
  }

  Future<void> delete<T>(PreferenceKey<T> key) async {
    if (key.storage == PreferenceStorage.secure) {
      await secure.delete(key: key.key);
    } else {
      await shared.remove(key.key);
    }
  }
}

class PreferencesService {
  final PreferencesStorageRouter router;

  PreferencesService(this.router);

  Future<T?> get<T>(PreferenceKey<T> key) async {
    final value = await router.read(key);
    return value ?? key.defaultValue;
  }

  Future<void> set<T>(PreferenceKey<T> key, T value) async {
    await router.write(key, value);
  }

  Future<void> remove<T>(PreferenceKey<T> key) async {
    await router.delete(key);
  }
}

Streams / reactive prefs

Чтобы слушать изменения:

Stream<T> watch<T>(PreferenceKey<T> key)

твоя задача разработать пакет, который будет предоставлять такой функционал, а также написать документацию и примеры использования. Добавить PreferenceKey serializer 

abstract class PrefSerializer<T> {
  String encode(T value);
  T decode(String value);
} 
от него можно будет наследоваться и создавать свои сериализаторы для сложных типов данных, например:
class DateTimeSerializer extends PrefSerializer<DateTime> {
	@override
	String encode(DateTime value) => value.toIso8601String();

	@override
	DateTime decode(String value) => DateTime.parse(value);
}
также сделать базовые сериализаторы для DateTime, List<String>, Map<String, String> и т.д.

и сама главная функция кодогенерация, которая будет генерировать код на основе описанных ключей и сериализаторов, чтобы разработчики могли использовать типизированные методы для доступа к своим предпочтениям.

пример 

Потенциальный API (идеальный)
@Prefs()
class AppPrefs {

  @Pref(defaultValue: ThemeMode.system)
  static const theme = PrefKey<ThemeMode>();

  @Pref(protected: true)
  static const vaultKey = PrefKey<String>();

}

Использование:

prefs.theme
prefs.theme = ThemeMode.dark

требования к коду должен быть чистым и разбитым на модули, хорошо структурированным и легко поддерживаемым. Кодогенерация должна быть реализована с помощью пакета build_runner и source_gen. Документация должна быть подробной и включать примеры использования. Пакет должен быть опубликован на pub.dev и иметь хорошую поддержку сообщества.
